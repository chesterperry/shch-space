## Процесс
На сборку контента и разработку у меня ушло около 60 часов работы, для этого потребовалось предварительно отрисовать макет в Figma и полностью переделать стартовую тему Gatsby, в процессе появлялись идеи для воплощения как технического, так и художественного плана, и конечный результат сильно отличается от первоначальной задумки отрисованной в макетах. 

## Цель
Целью проекта было собрать интерактивное портфолио по своей работе, закрепить знания о движке Gatsby, улучшить навыки работы с версткой и научиться распределять контент по всему сайту

## Стиль
Графический стиль, вдохновленный <LinkSimple to={"https://brutalistwebsites.com/"}>бруталистской эстетикой</LinkSimple> и технической документацией, представлен системой модульных сеток, яркими функциональными элементами и простой типографикой. Этот стиль отражает мое виденье и подход к работе.

## Архитектура
Мне нравится возможность переиспользования компонентов логику которых я понимаю, поэтому большая часть компонентов была написана с нуля, сайт состоит из index страницы, для страниц описания проектов и записей были написаны отдельные шаблоны

## Технологии
Использован движок статичных сайтов Gatsby, он состоит из ряда известных библиотек:
- React отвечает за мгновенное отображение контента и взаимодействие с интерфейсом,
- Tailwind CSS предлагает внутреннюю структуру классов для продуктивного описания элементов интерфейса,
- GraphQL предоставляет упрощенную и гибкую дистрибуцию контента по всем компонентам сайта во время сборки,
- Sharp обеспечивает скоростную обработку и сжатие изображений во время сборки,
- дополнительные плагины помогают получать контент из репозитория с исходным кодом и работать с удаленными источниками, например здесь использован плагин для работы с Google Sheets
- webpack собирает сайт после всех манипуляций с данными, контентом, упаковывает это и приводит в вид готовый для загрузки на хостинг

Здесь используется хостинг Github Pages с автоматической сборкой после каждого обновления кода в удаленном репозитории
